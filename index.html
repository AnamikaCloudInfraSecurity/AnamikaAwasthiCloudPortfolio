AWS Serverless Cloud Security

Enterprise-Ready Cloud Security Audit & Monitoring System

Designed and built a serverless security audit platform to dynamically inspect AWS resource configurations against policy rules, capturing and alerting on misconfigurations.


  • Scans IAM configuration states (no MFA, active keys without rotation, over-privileged roles).
  • Inspects network resource posture (open security groups, unencrypted S3 buckets).
  • Aggregates logs for historical audit validation.

A serverless infrastructure designed for high scalability, low cost, and edge performance:


  • Core Logic: AWS Lambda schedules scan operations via EventBridge schedules.
  • Alert Pipeline: Event alerts trigger AWS Simple Email Service (SES) logs.
  • Dashboard Hosting: Secure frontend deployed on private Amazon S3, distributed globally via CloudFront CDN utilizing secure Origin Access Control (OAC).

Overcame critical runtime constraints and design trade-offs:


  • Timeout Limits: Handled multi-account scan overhead by decomposing executions into concurrent, modular Lambda functions.
  • IAM Policies: Managed cross-account audit permissions using secure assume-role structures.
  • CORS Resolution: Resolved API Gateway cross-origin response formatting controls.

AWS Security Zero Trust

Zero Trust Secure API Gateway Demonstration System

Constructed an authorization structure simulating Zero Trust secure request architectures, verifying caller identities and verifying request signatures on every access.


  • Assumes a "never trust, always verify" posture for all API communication.
  • Eliminates public internet exposures for background compute structures.
  • Enforces least privilege IAM controls on all backend resources.

High-security backend configuration integrating Amazon API Gateway and Lambda:


  • Authentication: API Gateway uses AWS IAM Authorizer configurations.
  • Request Signature: Clients construct and sign requests with AWS Signature Version 4 (SigV4).
  • Compute isolation: Lambda function execution permissions map exclusively through roles inside private VPC configurations.

Addressed complex security verification challenges:


  • SigV4 Generation: Standardized frontend library configurations to accurately compile hash validation values.
  • Role Mappings: Outlined granular trust policy rules to prevent credential escalations.
  • Deny-by-Default Validation: Validated security boundaries using automated tests generating intentionally failed request tokens.
